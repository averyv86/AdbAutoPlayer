# First Run Guide

Complete walkthrough of running macOS Resource Optimizer for the first time.

## Step 1: Initialize the System

Start with initialization to verify everything is set up correctly:

```bash
/macos-resource-optimizer:0-init
```

Expected output:
```
Initializing macOS Resource Optimizer v7.0...
✓ System check complete
✓ Dependencies verified
✓ Configuration loaded
Status: Ready
```

## Step 2: Run System Analysis

Analyze your current system state:

```bash
/macos-resource-optimizer:1-analyze
```

This command:
- Collects CPU metrics
- Measures memory usage
- Analyzes disk I/O
- Checks battery status
- Monitors thermal conditions
- Checks network performance

Expected output:
```json
{
  "status": "success",
  "execution_time_seconds": 1.8,
  "timestamp": "2024-01-15T10:30:00Z",
  "categories": {
```

## Step 3: Review Analysis Results

Examine the analysis to understand your system state:

- Green indicators: All systems normal
- Yellow indicators: Worth monitoring
- Red indicators: Need attention

## Step 4: Run Optimization

Apply optimizations based on analysis:

```bash
/macos-resource-optimizer:2-optimize
```

This will:
- Optimize CPU scheduling
- Free up memory
- Clean up temporary files
- Optimize power settings
- Adjust thermal management
- Optimize network settings

Expected output:
```json
{
  "status": "success",
  "optimizations_applied": 12,
  "estimated_improvement_percent": 18,
  "execution_time_seconds": 2.0
}
```

## Step 5: Monitor Results

Start monitoring for continuous optimization:

```bash
/macos-resource-optimizer:3-monitor
```

This initiates:
- Real-time metric collection
- Performance tracking
- Alert generation
- Data logging

## Step 6: Generate Report

Create a detailed report of changes:

```bash
/macos-resource-optimizer:4-report
```

Report includes:
- Before/after metrics
- Optimizations applied
- Performance improvements
- Recommendations

## Common First-Run Questions

**Q: Should I run optimization immediately?**
A: Yes, safe to run on first use. Start with analysis to understand baseline.

**Q: How long does optimization take?**
A: Typically 1.5-2.5 seconds depending on system size.

**Q: Can I undo optimizations?**
A: Most optimizations are reversible. See [Troubleshooting](/guides/troubleshooting).

**Q: How often should I run?**
A: Recommend daily or when performance degrades.
