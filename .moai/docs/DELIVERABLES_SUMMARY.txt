  - Error type identification and diagnosis

  - Detailed status report generation

  - Magisk not responding: Auto-restart, up to 2 retries
  - zygisk-status timeout: Run full diagnostics
  - Hook load failure: Identify error type and recover
  - Permission denied: Retry with su -c prefix
  - Module conflict: Disable conflicting modules

Status Outcomes:
  1. FULLY FUNCTIONAL ✓
     - All checks passed, hooks loaded, ready for use
  
  2. PARTIALLY FUNCTIONAL ⚠
     - Core functions work, some modules have issues
     - Recommend disabling conflicting modules
  
  3. DISABLED ❌ (but installed)
     - Zygisk module exists but OFF
     - Action: Enable module and re-verify
  
  4. NOT INSTALLED ❌
     - Zygisk module missing
     - Action: Install from Magisk repository
  
  5. NON-FUNCTIONAL ❌
     - Multiple errors or unrecoverable
     - Action: Major troubleshooting required

  - {project-root} - Auto-detected project root
  - {config_source}:key - YAML config file lookup
  - {installed_path} - Current workflow directory
  - {output_folder} - Configured output location
  - {date} - System date variable

  - workflow: Execute multi-step workflow file
  - exec: Run shell command directly
  - action: Perform named action
