INSTALLATION STEPS (5 minutes)
================================================================================

1. Download APK
   $ wget https://github.com/senzhk/ADBKeyBoard/releases/download/v1.7/ADBKeyBoard-v1.7.apk \
     -O bin/ADBKeyboard.apk

2. Install on device
   $ adb -s 127.0.0.1:5555 install -r bin/ADBKeyboard.apk

3. Enable input method
   $ adb -s 127.0.0.1:5555 shell ime enable com.android.adbkeyboard/.AdbIME
   $ adb -s 127.0.0.1:5555 shell ime set com.android.adbkeyboard/.AdbIME

4. Test
   $ python test_korean_input.py --test-all

5. Done! Ready for Korean input

# Simple one-liner
from adb_auto_player.device.adb import AdbController
controller = AdbController()
controller.input_text_unicode('서초동')  # Korean!

# Using manager directly
manager = controller.unicode_input
manager.input_korean('한국어 테스트')
manager.input_chinese('中文测试')
manager.input_japanese('日本語テスト')

# Batch operations
texts = ['이름', '주소', '전화번호']
manager.input_text_sequence(texts, delay_between=0.5)

# Game bot integration
class GameBot:

        time.sleep(0.3)
        self.adb.input_text_unicode(name)  # Korean name
        time.sleep(0.5)
        self.adb.d.click(540, 400)  # OK button

================================================================================
ARCHITECTURE
================================================================================

Standard "adb input text" Command:
  Problem: Cannot handle Unicode (Korean, Chinese, Japanese)
  Limitation: Limited to ASCII/Latin-1

ADBKeyboard Broadcast Solution:
  Method: am broadcast -a ADB_INPUT_TEXT --es msg '서초동'
  Benefits: Full Unicode support, simple protocol, no root needed
  Status: Battle-tested, production-ready

Integration:
  - Lazy loading (created on first use)
  - Seamless integration with AdbController
  - No breaking changes to existing code
  - Backward compatible
